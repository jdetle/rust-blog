Step-by-step integration

1. Google Tag Manager (recommended)

tagmanager.google.com

GTM is a tag hub: add GA4, Meta Pixel, and other tags from the GTM UI without deploying code. Use GTM or direct GA4—not both in code.

  1. Sign in with Google
  2. Create account (or use existing) → Create container → Web
  3. Copy the Container ID (format GTM-XXXXXX)
  4. Add to .env.local: NEXT_PUBLIC_GTM_ID=GTM-XXXXXX — or run bun run ingest:gtm / bun run ingest:gtm GTM-XXXXXX
  5. In GTM: add a Google Analytics: GA4 Configuration tag with your Measurement ID. Trigger: All Pages.

Custom events: Use sendGTMEvent from @next/third-parties/google:

import { sendGTMEvent } from "@next/third-parties/google";

sendGTMEvent({ event: "buttonClicked", value: "xyz" });

2. Google Analytics 4 (direct, when not using GTM)

analytics.google.com

  1. Sign in with Google
  2. Admin (gear icon) → Create property (or use existing)
  3. Data streams → Add stream → Web
  4. Enter your website URL (e.g. jdetle.com)
  5. Copy the Measurement ID (format G-XXXXXXXXXX)
  6. Add to .env.local: NEXT_PUBLIC_GA4_ID=G-XXXXXXXXXX
  7. Run: bun run ingest:ga4 or bun run ingest:ga4 G-XXXXXXXXXX

Implementation: Uses @next/third-parties GoogleAnalytics component. Page views and client-side navigations are automatic. Custom events: sendGAEvent('event', 'name', { key: 'value' }) from @next/third-parties/google.

3. Microsoft Clarity

clarity.microsoft.com

  1. Sign in with Microsoft account
  2. New project → enter your site URL (e.g. jdetle.com)
  3. Settings → Project settings → copy the Project ID (numeric string)
  4. Add to .env.local: NEXT_PUBLIC_CLARITY_ID=YOUR_PROJECT_ID
  5. Run: bun run ingest:clarity or bun run ingest:clarity [PROJECT_ID]

Export (optional): For analytics-ingestion, get a Clarity Export API token from Clarity export docs. Add CLARITY_EXPORT_TOKEN to your Container App env.

Implementation: AnalyticsProvider injects the Clarity script. Session recordings start automatically.

4. PostHog

app.posthog.com/signup

  1. Sign up (or use cargo run --bin setup-analytics for API signup)
  2. Create a project
  3. Project Settings → Project API Key → copy (format phc_...)
  4. Add NEXT_PUBLIC_POSTHOG_KEY to Vercel (frontend)
  5. Add POSTHOG_API_KEY to Azure Container App / .env (same key — used by analytics-ingestion)
  6. Run: bun run ingest:posthog or bun run ingest:posthog phc_...

Implementation: AnalyticsProvider loads PostHog with person_profiles: 'identified_only'. Events and session replay are automatic.

5. Plausible

plausible.io ($9/mo)

  1. Sign up → Add website
  2. Enter your domain (e.g. jdetle.com)
  3. The domain is the env value — no API key needed for the script
  4. Add to .env.local: NEXT_PUBLIC_PLAUSIBLE_DOMAIN=jdetle.com
  5. Run: bun run ingest:plausible or bun run ingest:plausible jdetle.com

Implementation: AnalyticsProvider loads Plausible script with data-domain. Page views are automatic.

6. Vercel Analytics

vercel.com

  1. Vercel dashboard → your project
  2. Settings → Analytics
  3. Enable Web Analytics (toggle on)

Implementation: No env vars. The @vercel/analytics component in app/layout.tsx auto-injects the script when deployed to Vercel.

7. Meta Pixel

business.facebook.com/events_manager

  1. Create or log into a Meta Business Suite account
  2. Go to Events Manager (or Meta Business Suite → Data sources)
  3. Add new data source → Meta Pixel → Connect
  4. Name your pixel (e.g. "jdetle.com")
  5. Enter your website URL
  6. In the setup options, choose Install code manually
  7. Copy your Pixel ID (15–16 digit numeric string)
  8. Add to .env.local: NEXT_PUBLIC_META_PIXEL_ID=1234567890123456
  9. Add to Vercel: Project Settings → Environment Variables → NEXT_PUBLIC_META_PIXEL_ID

Note: Meta Pixel sets cookies and tracks users; ensure compliance with consent requirements (e.g. GDPR, CCPA) in your jurisdiction before enabling.

Implementation: Add this to components/analytics-provider.tsx:

const META_PIXEL_ID = process.env.NEXT_PUBLIC_META_PIXEL_ID ?? "";

{/* Meta Pixel */}
{!isPlaceholder(META_PIXEL_ID) && (
  <>
    <Script id="meta-pixel" strategy="afterInteractive">
      {`
        !function(f,b,e,v,n,t,s)
        {if(f.fbq)return;n=f.fbq=function(){n.callMethod?
        n.callMethod.apply(n,arguments):n.queue.push(arguments)};
        if(!f._fbq)f._fbq=n;n.push=n;n.loaded=!0;n.version='2.0';
        n.queue=[];t=b.createElement(e);t.async=!0;
        t.src=v;s=b.getElementsByTagName(e)[0];

        'https://connect.facebook.net/en_US/fbevents.js');
        fbq('init', '${META_PIXEL_ID}');

      `}

    <noscript>

        src="https://www.facebook.com/tr?id=${META_PIXEL_ID}&ev=PageView&noscript=1" alt="" />

)}

Custom events: To track conversions (e.g. newsletter signup, purchase):

// In your component, after a successful action:
if (typeof window !== "undefined" && (window as any).fbq) {
  (window as any).fbq("track", "Lead");  // or "CompleteRegistration", "Purchase", etc.
}

Standard events: PageView, ViewContent, Lead, CompleteRegistration, Purchase, AddToCart, InitiateCheckout. See Meta Pixel events reference.

Ingest script: Create scripts/ingest-env/meta-pixel.ts if you want a CLI helper (similar to ga4.ts).

Adding secrets to Vercel

  1. Vercel dashboard → Project → Settings → Environment Variables
  2. Add each NEXT_PUBLIC_* var. Apply to Production, Preview, and Development.
  3. Or CLI: vercel env add NEXT_PUBLIC_GA4_ID (paste when prompted). Ingest scripts accept --no-vercel to skip Vercel push.
  4. Use bun run ingest:vercel-sync to push all NEXT_PUBLIC_* from .env.local to Vercel.

Optional: Cosmos & analytics-ingestion

Used by the who-are-you event history (aggregated events from Clarity, PostHog, etc.).

  • Analytics API URL: Deploy analytics-ingestion to Azure Container Apps. Copy the URL. Run bun run ingest:analytics-api [URL].
  • Cosmos DB: Run cargo run --bin setup-analytics to provision via Azure CLI. It writes COSMOS_* to .env.
