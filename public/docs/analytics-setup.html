Step-by-step: getting each secret

Google Analytics 4

analytics.google.com

  1. Sign in with Google
  2. Admin → Create property (or use existing)
  3. Data streams → Add stream → Web
  4. Copy the Measurement ID (format G-XXXXXXXXXX)
  5. Run: bun run ingest:ga4 or bun run ingest:ga4 G-XXXXXXXXXX

Microsoft Clarity

clarity.microsoft.com

  1. Sign in with Microsoft account
  2. New project → enter your site URL (e.g. jdetle.com)
  3. Settings → Project settings → copy the Project ID (numeric string)
  4. For export (analytics-ingestion): Clarity Export API requires separate token — see Clarity export docs
  5. Run: bun run ingest:clarity or bun run ingest:clarity [PROJECT_ID]

PostHog

app.posthog.com/signup

  1. Sign up (or use cargo run --bin setup-analytics for API signup)
  2. Create a project
  3. Project API Key: Project Settings → Project API Key → copy (format phc_...)
  4. Add NEXT_PUBLIC_POSTHOG_KEY to Vercel (frontend)
  5. Add POSTHOG_API_KEY to Azure Container App / .env (same key — used by analytics-ingestion to pull events)
  6. Run: bun run ingest:posthog or bun run ingest:posthog phc_...

Plausible

plausible.io ($9/mo)

  1. Sign up → Add website
  2. Enter your domain (e.g. jdetle.com)
  3. The domain is the env value — no API key needed for the script
  4. Run: bun run ingest:plausible or bun run ingest:plausible jdetle.com

Vercel Analytics

vercel.com

  1. Project → Settings → Analytics
  2. Enable Web Analytics (no env vars — script is auto-injected)

Analytics API URL

Used by the who-are-you event history. Set to your Azure Container App URL after deploying analytics-ingestion.

  1. Deploy analytics-ingestion to Azure Container Apps (see deploy-analytics workflow)
  2. Copy the Container App URL (e.g. https://analytics-ingestion.xxx.azurecontainerapps.io)
  3. Run: bun run ingest:analytics-api or bun run ingest:analytics-api [URL]

Azure Cosmos DB (analytics-ingestion)

Run cargo run --bin setup-analytics to provision Cosmos DB via Azure CLI. It writes COSMOS_* to .env.

For Container Apps, add the same vars as environment variables (or Key Vault references).

Adding secrets to Vercel

  1. Vercel dashboard → Project → Settings → Environment Variables
  2. Or CLI: vercel env add NEXT_PUBLIC_GA4_ID (paste when prompted). Ingest scripts accept --no-vercel to skip Vercel push.
  3. Use bun run ingest:vercel-sync to push all NEXT_PUBLIC_* from .env.local to Vercel
